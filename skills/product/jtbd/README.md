# jtbd

> **Status:** research-preview • **Owner:** @jonathankv • **v0.1.0**

## What this does

Apply Jobs to Be Done framework để hiểu customers thực sự "hire" sản phẩm của mình để làm job gì. Interview design, forces analysis (F1-F4), job statements, demand-side competitive lens.

Dựa trên: Bob Moesta method + Anh Công Level 1/2 framework + real MoMo case evidence.

## Why this exists

Feature requests ≠ jobs. User feedback ≠ jobs. JTBD là lens duy nhất giúp phân biệt "customer muốn drill" vs "customer cần cái lỗ". Skill này gate PM khỏi build wrong thing.

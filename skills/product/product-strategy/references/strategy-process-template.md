# Strategy Process Template

Based on Chandra Janakiraman's operator's guide + Ravi Mehta's Strategy Stack + Ebi Atawodi's Vision framework.

---

## When to Use

- Mode A (Strategy Advisor): Full strategy formulation
- Mode C (Vision Architect): Vision + positioning
- Mode F (Strategy Decomposer): Breaking down complex strategic questions

---

## Phase 0: Pre-Flight (1-2 weeks)

### Leader Interviews ("Do You Even Like Fruits?")
Before building strategy, interview key stakeholders:

```
Questions to ask each leader:
1. What does success look like in 2 years?
2. What does failure look like?
3. What are the top 3 constraints we must respect?
4. What are you most worried about?
5. What should we STOP doing?
```

**Output:** Alignment map — where leaders agree, where they diverge.

### Strategy Stack Diagnosis
Check each layer. Debug bottom-up:

```
Goals not met?        → Check Roadmap
Roadmap unclear?      → Check Product Strategy
Strategy vague?       → Check Company Strategy
Company strategy?     → Check Mission
Mission unclear?      → Founder/CEO needs to articulate
```

---

## Phase 1: Preparation (4 weeks)

### User Problem Foundation
Build "Top 10 Things You Should Know" — living document of top 10 user problems.

```
For each problem:
- Problem statement (1 sentence)
- Evidence (data + qualitative)
- Current solution (how users cope today)
- Impact if solved (revenue, retention, satisfaction)
- Confidence level (high/medium/low)
```

### Market & Competitive Analysis
- Map competitive landscape using 7 Powers
- Identify: which competitors have which powers?
- Identify: where is power available but unclaimed?

### Winning Aspiration
Team exercise: imagine 2-year-out newspaper headline about your success.
> Example: "MoMo Becomes Vietnam's #1 Financial Super App with 50M Monthly Active Users"

---

## Phase 2: Strategy Sprint (1 week, cross-functional)

### Day 1: Problem Generation
- Each team member writes problems independently (Post-its / Figjam)
- No solutions allowed. Only problems.
- Cluster into 10-15 groups

### Day 2: Opportunity Framing + Ranking
For each cluster, frame as opportunity:
> "How might we [achieve X] for [user segment] by [approach]?"

Rank on 4 criteria (1-5 each):
1. **Expected Impact** — How much value if we nail this?
2. **Certainty of Impact** — How confident are we?
3. **Clarity of Levers** — Do we know what to build?
4. **Differentiation of Levers** — Can competitors easily copy?

**Select top 3 pillars** (focus areas). Everything else → explicit non-focus.

### Day 3: Anti-Focus Declaration
For each non-selected cluster, write:
> "We are NOT focusing on [X] because [Y]. We will revisit when [condition]."

This is the hardest part. Strategy without anti-focus is not strategy.

---

## Phase 3: Design Sprint (1 week)

### Visual Strategy
For each pillar, create:
- User journey map (current vs. desired state)
- Low-fidelity wireframes showing the strategic direction
- Success metrics (leading + lagging)

> "You would never work with an architect that didn't provide you a blueprint." — Ravi Mehta

### 7 Powers Check
For each pillar, assess:
- Which power does this build toward?
- What's the barrier to competitor imitation?
- If no power → reconsider the pillar

---

## Phase 4: Document Writing (1-2 weeks)

### Strategy Document Structure

```
1. Context (1 page)
   - Market dynamics, competitive landscape, user trends

2. Winning Aspiration (1 paragraph)
   - The newspaper headline + why it matters

3. Strategic Pillars (1 page per pillar, max 3)
   - Pillar name
   - User problem it addresses
   - Opportunity framing
   - Key bets (what we'll do)
   - Anti-bets (what we won't do)
   - Power it builds toward
   - Success metrics (leading + lagging + guardrail)

4. Non-Focus Areas (half page)
   - What we're NOT doing + why + revisit conditions

5. Risks & Pre-Mortem (half page)
   - Tigers (real threats)
   - Paper Tigers (seeming threats we're not worried about)
   - Elephants (unspoken concerns)

6. Roadmap Connection (1 page)
   - How pillars map to quarterly roadmap themes
   - Resource allocation across pillars
```

### Quality Check
- [ ] Does every pillar link to a user problem?
- [ ] Does every pillar build toward a specific power?
- [ ] Are non-focus areas explicit?
- [ ] Can team articulate strategy in 30 seconds?
- [ ] Does strategy force choices? Or is it a wish list?

---

## Phase 5: Rollout (2-3 weeks)

### Evangelism (Concentric Circles)
1. **Core team** (strategy working group) — already aligned
2. **Extended team** (engineering, design, data) — present + Q&A
3. **Leadership** (CEO, C-suite) — present + get buy-in
4. **Broader org** — all-hands, simplified version

### Living Document
- Revisit quarterly (not annually)
- Track: are we executing against pillars or drifting?
- Update non-focus areas as conditions change

---

## Small-S vs. Big-S Decision Guide

| Factor | Small-S | Big-S |
|--------|---------|-------|
| Horizon | 1-2 years | 5-10 years |
| Direction | Present forward | Future backward |
| Duration | 8-12 weeks | Up to 6 months |
| Frequency | Annual | Every 3-5 years |
| Trigger | Execution stalling, goals unclear | Market shift, new technology wave, company growth stage change |
| Output | Strategic pillars + roadmap | Vision + mission + strategic positioning |

## Vision Creation (Mode C specific)

### Step 1: Empathize
- Deep-dive into user problems (Top 10 list)
- Competitive landscape analysis
- Technology trend assessment

### Step 2: Create
Choose one approach:

**Mad Libs Template:**
> "Once upon a time, [current user problem]. And because of that, [consequence]. And one day, [our product vision]. And as a result, [transformed user experience]."

**Newspaper Headline:**
> Write the TechCrunch headline + subtitle of your future launch. Mock it up in actual publication template.

**Visual Mockup:**
> Draw/sketch the future product, even low-fidelity.

### Step 3: Evangelize
- Start with core team (build internal champions)
- Expand outward in concentric circles
- Use consistent language (same phrases, same story)

### Vision Quality Check (Four Pillars)
- [ ] Lofty — inspires the team?
- [ ] Realistic — attainable with effort?
- [ ] Beyond today's tech — not constrained by current limitations?
- [ ] Grounded in user problem — clear, potent problem at the center?
- [ ] Longevity — lasts 3+ years without annual reset?
